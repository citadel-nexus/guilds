## Usage Examples

### Python (boto3)

```python
import boto3
import json
import os

# Initialize Bedrock client
client = boto3.client(
    service_name='bedrock-runtime',
    region_name=os.getenv('AWS_BEDROCK_REGION', 'us-west-2'),
    aws_access_key_id=os.getenv('AWS_BEDROCK_ACCESS_KEY_ID'),
    aws_secret_access_key=os.getenv('AWS_BEDROCK_SECRET_ACCESS_KEY')
)

# Make API call
response = client.invoke_model(
    modelId='us.anthropic.claude-opus-4-5-20251101-v1:0',
    contentType='application/json',
    accept='application/json',
    body=json.dumps({
        "anthropic_version": "bedrock-2023-05-31",
        "max_tokens": 4096,
        "messages": [
            {
                "role": "user",
                "content": "Hello, Claude!"
            }
        ]
    })
)

# Parse response
result = json.loads(response['body'].read())
message = result['content'][0]['text']
print(message)
```

### Python (Anthropic SDK)

```python
from anthropic import AnthropicBedrock
import os

# Initialize client with Bedrock
client = AnthropicBedrock(
    aws_access_key=os.getenv('AWS_BEDROCK_ACCESS_KEY_ID'),
    aws_secret_key=os.getenv('AWS_BEDROCK_SECRET_ACCESS_KEY'),
    aws_region='us-west-2'
)

# Create message
message = client.messages.create(
    model='us.anthropic.claude-opus-4-5-20251101-v1:0',
    max_tokens=4096,
    messages=[
        {"role": "user", "content": "Hello, Claude!"}
    ]
)

print(message.content[0].text)
```

### Node.js

```javascript
import { BedrockRuntimeClient, InvokeModelCommand } from "@aws-sdk/client-bedrock-runtime";

// Initialize client
const client = new BedrockRuntimeClient({
  region: process.env.AWS_BEDROCK_REGION || "us-west-2",
  credentials: {
    accessKeyId: process.env.AWS_BEDROCK_ACCESS_KEY_ID,
    secretAccessKey: process.env.AWS_BEDROCK_SECRET_ACCESS_KEY
  }
});

// Prepare request
const payload = {
  anthropic_version: "bedrock-2023-05-31",
  max_tokens: 4096,
  messages: [
    { role: "user", content: "Hello, Claude!" }
  ]
};

const command = new InvokeModelCommand({
  modelId: "us.anthropic.claude-opus-4-5-20251101-v1:0",
  contentType: "application/json",
  accept: "application/json",
  body: JSON.stringify(payload)
});

// Invoke model
const response = await client.send(command);
const result = JSON.parse(new TextDecoder().decode(response.body));
console.log(result.content[0].text);
```

## Best Practices

### Security

1. **Never hardcode credentials** - Always use environment variables
2. **Rotate keys regularly** - We will notify you when rotation is needed
3. **Monitor usage** - Review your monthly usage reports
4. **Report suspicious activity** - Contact us immediately if you suspect unauthorized access

### Performance

1. **Implement retry logic** - Use exponential backoff for rate limit errors
2. **Batch requests** - Combine multiple queries when possible
3. **Cache responses** - Store frequently requested responses
4. **Use streaming** - For long responses, enable streaming for better UX

### Cost Optimization

1. **Right-size tokens** - Request only the tokens you need
2. **Optimize prompts** - Clear, concise prompts use fewer tokens
3. **Consider alternatives** - Use Sonnet for simpler tasks (we can provide additional keys)
4. **Monitor usage** - Review monthly reports for optimization opportunities

## Troubleshooting

### Issue: "AccessDeniedException"

**Cause**: Invalid credentials or insufficient permissions

**Solution**:
1. Verify credentials are set correctly in environment variables
2. Check for typos in access key ID or secret key
3. Ensure you're using `us-west-2` region
4. Contact Citadel support if issue persists

### Issue: "ThrottlingException" or Rate Limit Errors

**Cause**: Exceeded 60 requests/minute limit

**Solution**:
1. Implement request throttling in your application
2. Add exponential backoff retry logic
3. Contact us for higher capacity if needed

### Issue: "ModelNotFoundException"

**Cause**: Incorrect model ID or model not available in region

**Solution**:
1. Verify model ID: `us.anthropic.claude-opus-4-5-20251101-v1:0`
2. Ensure using `us-west-2` region
3. Check AWS Bedrock status page

### Issue: High Costs

**Cause**: Excessive token usage

**Solution**:
1. Review your prompt engineering
2. Reduce max_tokens when possible
3. Cache common responses
4. Review monthly usage report with us
