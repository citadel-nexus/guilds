### 2. Post-Call Distillation Loop

Derived from Sentinel's post-call memory storage pattern. Execution order:
1. **Save transcript** — `save_call_transcript()` persists to `call_transcripts` with `phone_number` direct column + `elevenlabs_conversation_id` upsert
1. **Fetch recent transcripts** — `get_recent_transcripts()` pulls last 5 transcripts by lead_id
1. **Inline-inject** — if DB missed the just-finished call, inject from request body
1. **Bedrock distillation** — `distill_context()` compresses history into compact DV JSON
1. **Save cache** — `update_context_cache()` upserts to `caller_context_cache`
1. **Publish NATS** — `citadel.voice.context.distilled` event
**DISTILL_PROMPT** explicitly extracts:
- `relationship_summary` — 2–3 sentences for next call
- `last_topics` — 3–5 topic tags
- `open_items` — bullet list of ALL commitments, promises, unresolved action items, quotes promised, follow-ups agreed to, discounts discussed
- `sentiment_trend` — cold → warming → warm → hot → customer → at_risk → churned
- `recommended_action` — what the agent should do/say first on the next call

### 3. Schema Migration — Root Cause Fix


[CALLOUT] 🔴 **Root cause discovered (Feb 26 Session 4):** The `call_transcripts` table (created by `20260217_secure_deal_rooms.sql`) used `transcript JSONB` and `call_duration_secs`, but `citadel_crew_api.py` queried `transcript_text TEXT` and `phone_number` — neither column existed. Every `get_recent_transcripts()` call returned 400, transcripts list was always empty, Bedrock never fired, cache never updated, and the agent always got cold defaults.

**Migration:** `20260226_call_transcripts_phone_column.sql`
```sql
-- Add phone_number column for direct pre-call lookup
ALTER TABLE public.call_transcripts
  ADD COLUMN IF NOT EXISTS phone_number TEXT;

-- Auto-extract text from JSONB transcript field
ALTER TABLE public.call_transcripts
  ADD COLUMN IF NOT EXISTS transcript_text TEXT
  GENERATED ALWAYS AS (transcript->>'text') STORED;

-- Indexes for fast lookup
CREATE INDEX IF NOT EXISTS idx_call_transcripts_phone
  ON public.call_transcripts(phone_number);
CREATE INDEX IF NOT EXISTS idx_call_transcripts_created_desc
  ON public.call_transcripts(created_at DESC);
```
**Additional fixes:**
- `PostCallRequest` model expanded — accepts full ElevenLabs post-call payload
- `get_recent_transcripts` — corrected column names, added JSONB fallback
- `distill_context` — references `call_duration_secs` (not `duration_seconds`)
- `save_call_transcript` — writes `phone_number` as direct column

### Central Health Aggregator (Phase 2.0)

```python
# tools/health_check_phase2.py
import httpx, asyncio
from datadog import statsd
import posthog

PHASE2_SERVICES = {
    "crew_api":    {"url": "http://localhost:8422/health", "critical": True},
    "mcp_server":  {"url": "http://localhost:8420/health", "critical": True},
    "mcp_fastapi": {"url": "http://localhost:8443/health", "critical": False},
    "college":     {"url": "http://localhost:8200/api/v1/college/health", "critical": False},
}

async def check_service(name: str, cfg: dict) -> dict:
    try:
        r = await httpx.AsyncClient().get(cfg["url"], timeout=10)
        data = r.json()
        status = data.get("status", "unknown")
        latency = r.elapsed.total_seconds() * 1000
        # Emit to Datadog
        statsd.gauge(f"citadel.phase2.{name}.healthy", 1 if status == "healthy" else 0)
        statsd.histogram(f"citadel.phase2.{name}.health_latency_ms", latency)
        # Emit to PostHog
        posthog.capture("system", f"phase2_health_check", {
            "service": name, "status": status, "latency_ms": latency,
            "critical": cfg["critical"], "phase": "2.0",
        })
        return {"service": name, "status": status, "latency_ms": latency}
    except Exception as e:
        statsd.gauge(f"citadel.phase2.{name}.healthy", 0)
        return {"service": name, "status": "unhealthy", "error": str(e)}

async def run_phase2_health():
    results = await asyncio.gather(*[check_service(n, c) for n, c in PHASE2_SERVICES.items()])
    critical_ok = all(r["status"] == "healthy" for r in results if PHASE2_SERVICES[r["service"]]["critical"])
    return {"phase": "2.0", "overall": "healthy" if critical_ok else "degraded", "services": results}
```

1. **Use **`**SUPABASE_SERVICE_KEY**` (not `SUPABASE_SERVICE_ROLE_KEY`) — `workspace.env` uses this non-standard name. `vault_loader.py` handles the mapping, but any new scripts or modules MUST use `SUPABASE_SERVICE_KEY` or go through `vault_loader`.
1. **Scope all NATS subjects to **`**citadel.voice.***` — Phase 2.0 voice events live under this namespace only. Do not publish to `citadel.crew.*` (reserved for Phase 3.0 guild build-out) or `citadel.sentinel.*` (Sentinel-only).
1. **Keep Bedrock distillation under 1,500 tokens** — The `DISTILL_PROMPT` must enforce compact output. If `prior_context` exceeds 1,500 tokens, the pre-call webhook will breach the 200ms latency target.
1. **Write to **`**caller_context_cache**`** via upsert only** — Never `INSERT` into cache; always `UPSERT` on `phone_number`. Duplicate cache rows cause the pre-call lookup to return stale data.

1. **Run guardrail scan on every commit** — All 16 guardrail rules from the CRP v1.0 protocol apply. Critical violations (`NO_FLOAT_MONEY`, `NO_SELECT_STAR`, `RLS_ENABLED`, `TENANT_ID_REQUIRED`, `STRIPE_ACCOUNT_HEADER`) block deployment.

### 🚫 MUST-NOT-DOs

1. **DO NOT use **`**src/main.py**`** pre-call route in production** — `crew_onboarding_routes.py` is a reference implementation only. Production runs `tools.citadel_crew_api:app` on port 8422 via systemd. Using the wrong entry point means zero DV injection.
1. **DO NOT modify **`**caller_identity**`** schema** — This table is shared across all voice agents (Sentinel, CREW, ZEA, future guild agents). Schema changes require a cross-team migration plan.

1. **DO NOT use **`**float**`** for any monetary values** — All currency amounts must be `INTEGER` cents or `NUMERIC(12,2)`. This applies to Stripe amounts, quote values, and the `last_quote_amount` DV.
1. **DO NOT store secrets in code or **`**.env**`** files committed to Git** — Use Supabase Vault + AWS Secrets Manager. The `workspace.env` is gitignored for a reason.
1. **DO NOT modify platform-level tables** — `tenants`, `tenant_members`, `auth.users` are off-limits. Phase 2.0 only touches `caller_identity`, `caller_context_cache`, and `call_transcripts`.
1. **DO NOT run College or Book Maker professors against CREW voice transcripts directly** — Transcripts must go through Bedrock distillation first. Raw transcripts contain PII (phone numbers, names, business details) that must not enter the professor chain unfiltered.

### Data Boundary Rules

1. **VCC → College:** Only distilled context (never raw transcripts). PII-scrubbed via `college_pii_scrubber.py`
1. **VCC → Book Maker:** Only lore fragments (structured JSON, not free text). Extracted via `vcc_lore_extractor.py`
1. **College → VCC:** Enrichment results only (`college_enrichment` JSONB blob). No professor chain metadata
1. **Book Maker → VCC:** Publication status only (`book_session_id`, `status`). No chapter content fed back to voice agents
1. **College ↔ Book Maker:** No direct connection in Phase 2.0. College enriches caller context; Book Maker consumes lore fragments. They do not talk to each other

### Failure Isolation

| If This Fails | Impact | Fallback |
| College :8200 down | No enrichment DVs; pre-call still works with cached context | Skip `college_enrichment` DV, serve from `caller_context_cache` only |
| Book Maker pipeline stalls | Lore fragments accumulate but no book compiled | Fragments stored in Supabase; compilation retries on next threshold trigger |
| NATS JetStream down | No async processing (enrichment, lore, feedback loop) | Post-call still saves transcript + cache locally; NATS replays on recovery |
| Bedrock distillation fails | No cache update, no downstream processing | Serve stale cache; log `VCC-QUESTION` flag; do NOT inject raw transcript |
| PII scrubber fails | College enrichment blocked | Hard block — never send unscrubbed data to professors |
