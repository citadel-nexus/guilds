## Audit Framework


### Audit Dimensions

Each guild is scored across **7 dimensions** on an A–F scale:
| **Dimension** | **Weight** | **What's Checked** | **Pass Threshold** |
| 🏗️ **Infrastructure** | 20% | ECS tasks running, health checks green, auto-scaling configured, Docker images in ECR | ≥ B |
| 📊 **Data Layer** | 15% | Supabase tables deployed, RLS policies enforced, indexes in place, migrations applied | ≥ B |
| 📡 **Event Mesh** | 15% | NATS subjects publishing/subscribing correctly, JetStream streams configured, no auth violations | ≥ B |
| 💳 **Monetization** | 15% | Stripe products configured, guild metadata set, entitlements enforced, payment flows tested | ≥ C |
| 🖥️ **UI/UX** | 10% | Landing page renders, guild branding applied, navigation correct, mobile responsive | ≥ C |
| 🔒 **Security** | 15% | Nemesis L6 compliance, tenant isolation, API auth, rate limiting, RBAC enforcement | ≥ B |
| 🔗 **Cross-Guild** | 10% | Inter-guild NATS events propagate, data flows verified, no orphaned events | ≥ C |

### Scoring Scale

| **Grade** | **Score** | **Meaning** |
| A | 90–100 | Production-ready, fully operational |
| B | 75–89 | Functional with minor gaps |
| C | 60–74 | Partially functional, needs work |
| D | 40–59 | Significant gaps, not shippable |
| F | 0–39 | Not deployed or critically broken |

## Audit 5 — Observability, Health Checks & SLA Verification


### 5.0 Health Check Audit Gate


[CALLOUT] 🩺 **Before any guild enters Phase 4.0 auditing, its health check endpoint must have been green for ≥ 72 consecutive hours.** This is a hard prerequisite — no exceptions.

```python
# tools/audit_health_gate.py
import httpx, asyncio
from datetime import datetime, timedelta
from datadog_api_client import ApiClient, Configuration
from datadog_api_client.v1.api.monitors_api import MonitorsApi

HEALTH_GATE_HOURS = 72  # Minimum consecutive green hours

async def verify_health_gate(guild: str, monitor_id: int) -> dict:
    """Check Datadog monitor history for consecutive uptime."""
    config = Configuration()
    with ApiClient(config) as api_client:
        api = MonitorsApi(api_client)
        # Get monitor state history for past 72h
        now = datetime.utcnow()
        history = api.get_monitor_sli_history(
            monitor_id,
            from_ts=int((now - timedelta(hours=HEALTH_GATE_HOURS)).timestamp()),
            to_ts=int(now.timestamp()),
        )
        uptime_pct = history.data.overall.sli_value
        passed = uptime_pct >= 99.9  # Allow ~4 min of blips in 72h
        return {
            "guild": guild,
            "monitor_id": monitor_id,
            "uptime_72h_pct": uptime_pct,
            "gate_passed": passed,
            "checked_at": now.isoformat(),
        }

async def run_health_gate_audit():
    GUILD_MONITORS = {
        "commerce": 100001, "intelligence": 100002, "research": 100003,
        "creator": 100004, "builder": 100005, "finance": 100006,
        "entertainment": 100007, "writer": 100008,
    }
    results = await asyncio.gather(*[
        verify_health_gate(g, m) for g, m in GUILD_MONITORS.items()
    ])
    return {"audit": "health_gate", "results": results, "all_passed": all(r["gate_passed"] for r in results)}
```
